Installation

The extension is loaded as an unpacked extension during development. Chrome Web Store submission is on the roadmap.

1
Clone the repository
git clone https://github.com/Sanskar-bot/Password-analyzer.git
The extension source is in the extension/ directory.
2
Open Chrome Extensions
Navigate to chrome://extensions in Chrome or Edge.
3
Enable Developer Mode
Toggle "Developer mode" in the top-right corner of the extensions page.
4
Load Unpacked
Click "Load unpacked" and select the extension/ folder from the cloned repository.
5
Test on any site
Navigate to any site with a password field (e.g. a login or signup page). The PassGuard widget will appear below the field automatically.
Technical Architecture

How it works

The extension uses a content script (content.js) that runs on every page. It watches for password <input> fields using a MutationObserver and injects an analyzer widget when it finds one.

The widget is rendered inside a Shadow DOM host element, ensuring complete CSS isolation — the widget never inherits or pollutes the host page's styles.

All analysis logic runs synchronously in the content script context. The shared modules from /shared/ are bundled directly — no message passing, no background worker, no server calls.

Privacy by Design
No network requests
The extension makes zero outbound network requests. There is no telemetry, no analytics, no CDN.
No password reading
The extension reads the password value only to analyze it locally. It never stores or transmits password values.
Shadow DOM isolation
The widget cannot access or modify the host page's DOM or JavaScript context beyond attaching its host element.
Minimal permissions
The manifest requests only activeTab and storage permissions — the minimum required to function.
